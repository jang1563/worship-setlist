# WorshipFlow UX 리뷰 보고서 (UX Review Report)

## 요약 (Executive Summary)

WorshipFlow는 **혁신적인 AI 기반 송리스트 생성 기능**을 핵심으로 하는 한국 교회 특화 도구입니다. 현재 구현된 MVP(최소 기능 제품)는 **핵심 가치 제안을 잘 전달**하고 있으나, **사용자 여정의 여러 지점에서 개선 기회**가 있습니다.

---

## 1. 사용자 페르소나별 UX 분석

### Persona 1: 신입 찬양 인도자 (25세, 1년차)

**현황 평가**

✅ **강점 (Strengths)**
- **초대화면의 명확한 CTA**: "AI 찬양 컨설턴트"로 용도를 즉시 이해
- **예시 메시지 제공**: 3가지 샘플 프롬프트가 진입 장벽을 낮춤
- **즉각적인 피드백**: AI 응답이 곡 리스트 형태로 명확하게 표시
- **직관적 드래그앤드롭 편집**: 곡 순서 변경이 쉬움
- **모바일 친화적**: 터치 디바이스 감지 후 TouchBackend 사용

❌ **약점 (Weaknesses)**

1. **곡 DB가 불충분함**
   - README에 "20곡 내장"이라고 했으나, 실제로는 374곡 보유
   - 곡 검색 시 원하는 곡이 없을 경우 대체 방안이 불명확

2. **AI 응답의 신뢰성 불안감**
   - 키 호환성 분석은 제공되지만, "자동 분석"이라는 표현이 모호함
   - 왜 특정 곡이 추천되었는지의 이유가 충분하지 않음
   - "데모 모드" 메시지가 자주 표시되어 신뢰도 저하

3. **키 전환 정보의 부재**
   - 키 호환성 체크는 있으나, **실제 악기 연주 가이드가 없음**
   - 피아노 브릿지를 어떻게 연주하는지 미상
   - "피아노 2마디 브릿지" 같은 텍스트만 제시되고 구체적 예시 없음

4. **설교 정보와 곡의 연결 약함**
   - AI가 설교 주제를 받아도 관련 말씀(scripture_ref) 매칭이 불명확
   - 신입 인도자는 왜 그 곡이 그 주제와 맞는지 확신이 서지 않음

5. **저장 및 불러오기 UX**
   - 저장 전에 "제목, 날짜, 예배 유형" 입력이 필수
   - 빈 송리스트 화면에서는 저장 버튼이 비활성화되어 혼란
   - 이전 송리스트를 "로드"하는 과정이 추가 클릭 필요

6. **에러 메시지의 친절함 부족**
   - "요청이 너무 많습니다" (429 에러) → 사용자 입장에서는 무엇을 해야 할지 모름
   - "죄송합니다. 오류가 발생했습니다" → 너무 일반적

7. **YouTube 링크의 불완전함**
   - 어떤 곡에 YouTube URL이 있는지 목록에서 미리 볼 수 없음
   - "YouTube URL이 없는 곡이 있습니다" 경고만 나타남

---

**개선 제안**

| 개선 항목 | 우선순위 | 구현 방법 |
|-----------|---------|----------|
| 곡 검색 시 추천/대체 곡 제시 | 높음 | "이 곡이 없습니다. 비슷한 곡:" 하단 추천 |
| 키 전환 가이드 확장 | 높음 | 각 악기별 연주 예시 영상 또는 음성 샘플 추가 |
| 설교 주제 ↔ 곡 매핑 명확화 | 중간 | 각 곡의 "성경적 의미" 페이지 추가 |
| 저장 UX 개선 | 중간 | 자동 저장 또는 빠른 저장 옵션 (기본값 입력) |
| 에러 메시지 개선 | 중간 | "API 요청이 초과되었습니다. 1분 후 다시 시도해주세요." |
| YouTube URL 표시 | 낮음 | 곡 목록에 작은 아이콘 표시 |

### Persona 2: 베테랑 찬양팀 리더 (35세, 10년차)

**현황 평가**

✅ **강점**

- **고급 기능들이 풍부함**: 팀, 공유, 내보내기 등 레더용 기능 구현
- **다양한 내보내기 옵션**: PDF, PowerPoint, 텍스트
- **키 흐름 분석**: "자연스러움/괜찮음/어색함" 3단계 평가가 직관적
- **송리스트 저장/로드**: 여러 송리스트 관리 가능
- **드래그앤드롭 편집**: 팀 간 신속한 조정 가능

❌ **약점**

1. **팀 기능이 미흡함**
   - 팀 기능이 있으나, **팀원들과 실시간 공동 편집이 불가능**
   - 송리스트 공유 → 링크 복사만 가능하고, 직접 편집 불가
   - 팀원 간 의견 수렴 방식이 불명확 (댓글, 투표 등 없음)

2. **연습 스케줄 관리 부족**
   - 찬양팀 리더는 "연습 일정 관리"를 필요로 하는데 기능 없음
   - 팀원별 참석 여부 추적 불가
   - 연습 진행률(어떤 곡까지 준비됐는지) 표시 불가

3. **팀원 악기별 할당 미흡**
   - team_members 테이블에 "instrument" 필드가 있지만 UI에 미반영
   - 예: "피아노 담당: 김수현, 기타 담당: 이선영" 같은 역할 할당 불가
   - 팀원의 음역대/능력을 저장할 방법 없음

4. **코드차트 관리 부족**
   - 코드 추출 기능은 있으나, **팀원들이 함께 검증하고 업데이트하는 워크플로우 없음**
   - chord_charts 테이블의 "contributor_id", "verified" 필드가 있지만 UI 없음
   - "커뮤니티 코드 입력" 기능이 언급되어 있으나 구현 안 됨

5. **트렌드/통계 대시보드 부재**
   - TrendsDashboard 컴포넌트가 있지만 실제 기능 미확인
   - 팀 리더가 원할 "최근 불린 곡", "선호도 분석" 등이 없음
   - usage_history 테이블이 있지만 분석 엔드포인트 없음

6. **권한 관리 부족**
   - teams.members에 "role" 필드 (admin, leader, member)가 있지만 UI 미구현
   - 누가 송리스트를 수정할 수 있는지 제한 불가

| 개선 항목 | 우선순위 | 구현 방법 |
|-----------|---------|----------|
| 실시간 공동 편집 (WebSocket) | 높음 | 팀원 초대 → 공동 작업 공간 (Collaborative editing) |

| 연습 체크리스트 | 중간 | "연습 준비도: 곡1(100%), 곡2(50%), 곡3(0%)" 대시보드 |
| 트렌드 분석 대시보드 | 중간 | "최근 3개월 인기곡", "팀 선호 장르" 등 |

### Persona 3: 소규모 교회 목사 (45세, IT 낮음)

**현황 평가**

✅ **강점**

- **간단한 첫 시작**: 채팅 인터페이스로 "말하기만 하면 됨"
- **한국어 UI**: 완전 한국어 지원으로 진입 장벽 낮음
- **예시 제공**: "새벽예배용 잔잔한 찬양 3곡" 같은 구체적 예시
- **안내 문구**: "이 추천은 참고용입니다..." 책임 회피 표현이 명확
- **설교 주제 반영**: "설교 주제는 '성령의 인도하심'이야" 같은 자연스러운 입력

❌ **약점**

1. **기술 용어의 과다 사용**
   - "키", "트랜스포즈", "브릿지", "코드", "ChordPro" 등 음악 용어
   - 일반 목사는 이런 용어를 이해하지 못할 가능성 높음
   - "내 팀이 G키를 선호해" 같은 입력을 할 수 없음

2. **모바일 UI의 완성도 낮음**
   - 목사는 주로 핸드폰으로 접근할 가능성 높음
   - 모달이 "bottom sheet"로 열려서 좋지만, 폰트가 작을 수 있음
   - 인쇄 미리보기가 데스크톱 중심

3. **프레젠테이션 기능 부족**
   - "프레젠터 모드"가 구현되어 있으나, 실제 기능이 불명확
   - 목사는 예배 중 가사를 프로젝터에 띄우고 싶은데, UI 단순함
   - 폰트 크기, 색상 커스터마이징 어려움

4. **예배별 템플릿 부족**
   - 새벽예배, 주일예배 등 템플릿이 PRD에는 나열되어 있으나, 실제 UI에 미반영
   - 목사가 매번 같은 정보(예배 유형, 시간 등)를 입력해야 함
   - 자주 사용하는 패턴 저장 불가

5. **말씀과의 연결 약함**
   - 설교 본문/주제를 입력받지만, AI 응답에 관련 성경 구절이 명확하지 않음
   - "로마서 8:1 관련" 같은 연결만 있고, 실제 성경 구절 검색 불가
   - 목사는 "부활의 승리"라는 주제로 부활절 찬양을 원하는데, 자동 매칭 어려움

6. **팀이 없을 경우 활용성 떨어짐**
   - 소규모 교회는 "팀"이 없을 수 있음 (목사가 혼자 결정)
   - 팀 기능이 강조되면 오히려 기능이 복잡해 보임
   - "개인 용" vs "팀용" 모드 구분 없음

## 2. 전체 앱 워크플로우 분석

### 현재 주요 사용자 여정

```
시작화면 (Chat)
  → 예배 정보 입력
  → AI 송리스트 생성
  → 송리스트 편집 (드래그앤드롭)
  → 저장
  → 인쇄/공유/내보내기
```

### 문제점

1. **모호한 다음 단계**
   - AI가 송리스트를 생성한 후, "이제 뭘 해야 하나?"가 불명확
   - "저장" vs "확인" vs "공유" 등 선택지가 많음
   - 새 사용자는 저장해야 하는지 모를 수 있음

2. **컨텍스트 손실**
   - 채팅 중 수정 요청 후 송리스트 편집기로 넘어가면 컨텍스트 손실
   - "아까 요청한 수정이 적용되었나?" 확인 어려움

3. **피드백 루프 부재**
   - 송리스트 저장 후 "이것이 정말 잘 구성되었나?" 피드백 받을 수 없음
   - AI가 "이 구성 어떤가요?" 물어보지 않음

## 3. AI 기능 평가

### 강점

✅ **생성 품질**
- 설교 주제를 잘 반영하여 곡 선정
- 역할 분류 (시작/경배/고백/선포/응답)가 명확
- 키 흐름 분석이 실시간으로 제공

✅ **UX**
- 채팅 형식으로 사용하기 쉬움
- 수정 요청을 자연스럽게 받아들임 ("10분으로 줄여줘" 등)

### 약점

❌ **신뢰성**
- "데모 모드"가 자주 표시되어 신뢰도 저하
- API 키 없을 때 품질이 현저히 떨어짐
- 같은 요청을 두 번 하면 다른 결과가 나올 수 있음 (재현성 낮음)

❌ **맥락 이해**
- 설교 본문을 제대로 분석하지 못함
- "로마서 8장" → 관련 곡들이 부정확할 수 있음
- 팀의 악기/수준을 고려하지 못함

❌ **제한사항**
- 곡 DB에만 의존하므로, DB가 제한적이면 추천이 제한됨
- 최신 CCM을 반영하지 못함
- 사용자 정의 곡(자작곡) 추가 불가

## 4. 강점 (Strengths) - 종합 평가

### 제품 레벨
✅ **혁신적 핵심 개념** - AI 대화형 송리스트 생성이 진정한 차별점
✅ **한국 특화** - CCM + 찬송가 + 번안곡 지원
✅ **사용성** - 입력 진입 장벽이 낮음 (채팅 형식)
✅ **오픈소스** - 무료 제공으로 작은 교회도 접근 가능

### 기술 레벨
✅ **반응형 UI** - 모바일 지원 (터치 디바이스 자동 감지)
✅ **상태 관리** - zustand로 간결한 상태 관리
✅ **에러 처리** - ErrorBoundary, 재시도 기능 포함
✅ **API 설계** - RESTful 구조, 명확한 스키마

---

## 5. 약점 (Weaknesses) - 종합 평가

### 심각한 문제 (Critical)

🔴 **곡 DB 부족** - 더 많은 곡 필요 (특히 최신 CCM)
🔴 **코드/악보 기능 미완성** - 실제 연습 도구로 부족함
🔴 **팀 협업 미흡** - 리더용 기능이 불완전

### 중간 문제 (Medium)

🟠 **음악 용어 설명 부족** - 일반 사용자에게 어려움
🟠 **프레젠테이션 기능 부족** - 예배 진행 어려움
🟠 **말씀 연결 약함** - 영적 깊이 낮음

### 경미한 문제 (Minor)

🟡 **UI 일관성** - 곳곳에서 모달 스타일이 다름
🟡 **로딩 피드백** - 느린 API 응답 시 사용자 혼란
🟡 **에러 메시지** - 너무 기술적이거나 일반적

## 6. 개선 기회 (Opportunities) - 우선순위별

## 7. UI/UX 구체적 개선안

### 개선안 1: 온보딩 개선

**현재**: 예시만 제시 → 사용자 입력 받음

**개선**:
```
1단계: "예배 유형 선택" (버튼: 새벽/주일/청년/특별)
2단계: "예상 시간" (슬라이더: 10-60분)
3단계: "설교 정보" (선택사항)
4단계: "팀 구성" (악기 선택: 피아노/기타/드럼/성악)
5단계: "AI 생성"
```

→ **효과**: 신입 사용자도 단계별로 쉽게 입력 가능

---

### 개선안 2: AI 응답 포맷 개선

**현재**:
```
1. 곡명 (Key: X) - 역할
2. 곡명 (Key: Y) - 역할
...
```

**개선**:
```
1️⃣ 곡명 (Key: X)
   ├ 역할: 시작/감사
   ├ 말씀: 로마서 8:1
   ├ 시간: 4분 30초
   └ 다음 곡과의 전환: 피아노 2마디 브릿지

2️⃣ 곡명 (Key: Y)
   ...

📊 흐름: G → A → D → G ✅ 자연스러움
💬 설명: 감사→경배→선포→응답의 자연스러운 진행
```

→ **효과**: 정보가 한눈에 들어오고, 왜 그렇게 구성했는지 명확

---

### 개선안 3: 프레젠터 모드 개선

**추가 기능**:
- 곡명, 가사(외부 링크), 키 표시
- 폰트 크기 조절 (기본 28pt → 최대 48pt)
- 배경색 선택 (검은색, 파란색, 흰색 등)
- 타이머 표시 (예배 진행 시간)
- QR 코드로 팀원과 연동

→ **효과**: 목사가 실제 예배에서 사용 가능

---

### 개선안 4: 팀 협업 UI

**추가**:
```
팀: [팀 선택하기] 또는 [혼자 사용]
↓
팀원 추가: 김수현(피아노, 음역대 High), 이선영(기타), 박준호(드럼)
↓
역할 할당: "이 송리스트에서 누가 리드 보컬?"
↓
체크리스트:
  ☐ 곡1 - 김수현 (실수 없음)
  ☐ 곡2 - 이선영 (음정 확인 필요)
  ☐ 곡3 - 박준호 (템포 확인 필요)
```

→ **효과**: 리더가 팀 준비 상황을 한눈에 파악

## 8. 특정 페르소나를 위한 맞춤 개선

### 신입 인도자를 위해

```
핵심: "이 앱을 써서 내가 무엇을 할 수 있나?"를 명확히
- 온보딩: 단계별 가이드
- 팁: "키란 음정입니다" 풍선 도움말
- 성공 메시지: "축하합니다! 첫 송리스트를 만들었어요!"
- 다음 단계: "이제 팀원들과 공유해보세요"
```

### 베테랑 리더를 위해

```
핵심: "팀원들과 효율적으로 협업하기"
- 팀 대시보드: 팀원 관리, 역할 할당, 준비도 추적
- 고급 기능: 코드 검증, 통계 분석
- 통합: Google Calendar, Slack 등 연동 고려
```

### 목사를 위해

```
핵심: "예배 준비를 간단하게"
- 빠른 시작: "새벽예배" 버튼 하나로 템플릿 로드
- 음악 용어 배제: "높은 음" (X) → "높은 음역대가 필요합니다" (O)
- 영적 맥락: 말씀 연결, 기도, 감사 요소 강조
```

- **응답 시간**: API 평균 응답 시간
- **코드 품질**: 테스트 커버리지, 린터 점수

## 10. 결론 및 권고사항

### 종합 평가

WorshipFlow는 **한국 교회를 위한 혁신적인 도구**이며, **핵심 가치는 충분히 전달**되고 있습니다.

**강점**:
- ✅ AI 기반 송리스트 생성 (차별화)
- ✅ 사용하기 쉬운 인터페이스
- ✅ 한국 교회 특화

**약점**:
- ❌ 곡 DB 확대 필요 (최신 CCM)
- ❌ 팀 협업 기능 미완성
- ❌ 프레젠테이션 기능 약함
